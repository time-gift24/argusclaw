# TinyVue 组件的使用

## 组件索引

所有的组件名称都可以从 [menus.js](../menus.js) 文件中获得。在 `cmpMenus` 变量中，存放的组件的目录，它包含着二级数据，第一级是组件的类型，第二级数据才是真正的组件信息。

第二层的每一项数据的`name` 记录着组件的名称， `namecN` 记录着组件的中文名称， `key` 记录着组件的小写连字符格式的组件名。通过 `key` 属性可以在`apis`中找到相应的组件 API 文档， 在`demos`中找到相应的组件示例的文件夹。

`cmpMenus`的数据格式示例：

```
 {
    'label': '基础',
    'labelEn': 'Basic',
    'key': 'cmp-frame-style',
    'children': [
      { 'nameCn': '按钮', 'name': 'Button', 'key': 'button' }]
 }
```

上述数据描述了：在`基础`类别的组件中`Button`组件，它的 key 是'button' 。

### 组件的 API 文档索引

在`../apis` 目录中，每一个文件对应一个组件的 API, 其文件名对应于组件英文名，比如： `action-menu.md` 就是对应着 `ActionMenu` 组件。

每一个 API 文件中，可能包含多个组件，比如`form.md` 中包含了 Form 和 FormItem 组件的文档。一个组件的 API 信息包含：

- `props`描述了组件的属性
- `events`描述了组件的事件
- `methods`描述了组件的实例上的方法
- `slots`描述了组件的插槽

在一些 API 文件中， 还有 `Types`声明，对应着文档中的某些属性或事件参数的类型。

API 示例：
[button 的 API 索引](../apis/button.md) 存放着`button`组件的 API 文档。

### 组件的示例索引

在`../demos`目录中，每一个文件夹对应一个组件的全部示例，其中文件名规则是：

- `${组件名}/webdoc/${组件名}.md` :组件的示例索引文件,其中`demos`属性存放着该组件所有的示例目录,记录着示例的说明和示例指向的示例源码。 每一个示例信息中, `demoId` 是示例的唯一 id, `名称`是示例的名字, `描述`是示例的详细说明, `代码文件` 记录着该示例的源码位置。
- `${组件名}/${demoId}.vue` Vue 的选项式语法的示例源码。
- `${组件名}/${demoId}-compossition-api.vue` Vue 的组合式语法的示例源码。

demos 示例：
[button 的示例索引](../demos/button/webdoc/button.md) 存放着`button`组件的所有示例。

## 严格查找流程，严禁跳过任意阶段

查找组件的 API 和示例时，要严格遵循下面的查找阶段：

1. **组件名查找阶段** ：从`../menu.js`中找到适合的组件名称
2. **查找 API 信息阶段**: 通过组件名称，在`../apis`中查找组件的 API 文档
3. **示例查找阶段**： 通过组件名称，在 `../demos` 先查找组件的示例索引文件，然后再根据示例文件中的 `codeFiles` 字段查找组件的示例源码。首先要查看`basic-usage.vue` 的基础用法，再查看其它的示例用法。

### 查找约束规则

1. 要根据组件名进行关联查询 API 和示例，严禁使用其它开源库的属性，来猜测 TinyVue 组件库的用法。
2. 读取属性描述或示例描述时，必须使用它的中文描述信息，通常是在 `zh-CN`的属性中描述的，一定要忽略`en-US`描述信息。
3. 查找不到相应的属性或示例时，不得自行推断或替换其它信息。
4. 遇到错误时，不得自行推断或替换其它信息。
5. 在查找 `icon` 图标时，必须从 `../demos/icon/iconGroups.js` 文件中查找正确的图标名称，禁止使用此文件之外的其他图标名称；`icon` 的使用方式一定要参考 `..demos/icon` 目录下的示例索引文件和示例源码，不要自行决定（和业界的 icon 使用方式不同 TinyVue 组件的 icon 是一个函数，需要调用才能使用）。
